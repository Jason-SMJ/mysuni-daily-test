## 6. 고위험 이슈

### 6.1 미탐 1건

- 시나리오: `career_profile`
- 항목: item 10 `프로젝트 팝업 노출 확인`
- AI 결과: `정상`
- 사람 결과: `비정상`
- 영향: 실제 팝업 본문이 잘리지 않고 떠야 하는데 화면 상단만 보이는 상태를 정상으로 통과시킴
- 원인 추정: 팝업 제목은 보여도 본문 확인 기준이 프롬프트에 약하게 정의됨

### 6.2 오탐 3건

- `career_profile` item 11: 이전 모달이 반투명 오버레이로 남아 정상 팝업이 비정상으로 판정됨
- `career_mypick` 메인: 추천 카드 하단 일부만 잘린 상태를 화면 오류로 과도 판정함
- `career_myprogress` 차트: 숫자 레이블이 희미하게 보이는 상태를 콘텐츠 미표시로 판정함

### 6.3 판단불가 13건

- `career_profile` popup 계열 5건
- `career_myprogress` 차트 영역 2건
- `career_mypick` 카드 리스트 2건
- 기타 4건

공통 원인:
- 캡처 범위가 넓어 배경과 팝업이 함께 포함됨
- 텍스트 가독성이 낮은 축소 뷰가 포함됨
- 로딩 직후 캡처되어 UI가 완전히 자리잡기 전에 판단 수행

## 10. 개선 액션

### 즉시 조치

- `career_profile` popup 계열 항목에서 요소 단위 캡처 우선 적용
- popup 검증 전후 `close_all_popups` 호출 강화
- 팝업 정상 기준에 `제목 + 본문 핵심 텍스트 + CTA 버튼` 확인 조건 추가

### 다음 배포 후보

- 항목 단위 결과를 JSONL로 저장
- Slack 전송 결과에 `message ts` 저장
- 판단불가 발생 시 재촬영 1회 정책 검토
- baseline 이미지 갱신 이력 관리 추가

## 11. 다음 주 집중 모니터링 항목

- `career_profile` item 10, 11 popup 검증 정확도

## 12. 운영 판정

이번 주 운영 상태는 `Yellow`입니다.

판단 근거:
- Daily 배치 자체는 안정적으로 수행됨
- Slack 누락은 없고 운영 알림 체계는 동작함
- 다만 `비정상 Recall`, `Precision`, `판단불가율`이 목표 미달
- 특히 popup 계열에서 미탐 1건이 발생해 운영 신뢰도 보완이 필요함

## 13. 요약 코멘트

현재 시스템은 `일일 자동 점검 도구`로서의 기본 운영은 가능하지만, `완전 자동 판정 시스템`으로 보기에는 popup 계열과 일부 동적 화면에서 여전히 보완이 필요합니다. 다음 주에는 popup 항목 정확도 개선과 항목 단위 결과 로그 저장을 우선 적용하는 것이 타당합니다.
